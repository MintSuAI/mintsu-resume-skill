# MintSu Resume · 薄荷简历（mintsu-resume）

写简历和改简历的全流程 skill。核心理念：**简历是论证，不是文档**——它是从素材库里长出来的，不是憋出来的。

## 铁律（每一步都生效）

1. **只换表述，永不虚构。** 每一条写进简历的内容，都必须是用户真实做过、面试敢被追问的。用户提供不了的事实，宁可留空也不编。
2. **讲不成数字的价值，等于没有。** 每条经历尽力追问出量化结果；确实没有数字的，量化范围/频次/规模。
3. **素材表只加不改**——它是用户职业生涯的账本，所有版本简历的唯一事实源。
4. 一页 A4。投行标准：每一行都要有数字，每一个字都要值钱。

## 使用流程（按用户所处阶段进入，不必从头走）

### 第一步：量化盘点 → 经历素材表
用户丢来旧简历/周报/项目记录，或直接口述。按 [references/01-盘点追问.md](references/01-盘点追问.md) 的追问协议逐条深挖，产出「经历素材表」（做了什么事 / 结果与数字 / 用到的技能），再从素材表长出一份通用简历母版。

### 第二步：按 JD 定制
用户给目标岗位 JD。按 [references/02-JD定制.md](references/02-JD定制.md)：提取 15-20 个关键词 → 判断该岗位的叙事主轴 → 从素材表重选、重排、重讲每条经历，关键词自然注入 summary/首条 bullet/技能区。同一段经历，投产品讲闭环、投运营讲增长——事实不变，讲法换。

### 第三步：生成 PDF
按 [references/03-生成PDF.md](references/03-生成PDF.md)：把定稿内容填入 templates/ 的 HTML 模板（默认「通用」版；金融向参考金融人示例、运营/市场向参考运营人示例），浏览器打开 → 打印 → 存为 PDF（A4、无页眉页脚、默认边距）。改任何内容只改 HTML 对应文本节点，版式不动。

## 模板清单
- `templates/简历模板-通用.html` — 占位符版，每个板块附写作公式提示
- `templates/简历示例-金融人-虚构.html` — 成品示范（券商研究员「李明远」，虚构）
- `templates/简历示例-运营人-虚构.html` — 成品示范（用户增长运营「王小雨」，虚构）
- 版式规格：A4 单页、深青 #0e7a6e 强调色、加分项徽章样式；示例人物均为虚构，页脚带教学演示声明，替换内容时记得移除该声明
