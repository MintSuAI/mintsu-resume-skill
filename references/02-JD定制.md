# 第二步：按 JD 定制

目标：同一套真实素材，按目标岗位讲出最大分量。**不存在"一份简历"，只存在"一套素材 + N 种讲法"。**

## 流程

### 1. 提取关键词（15-20 个）
通读 JD，分四类提取：
- 硬技能/工具（SQL、Excel、投放平台…）
- 业务域词（风控、增长、供应链、GTM…）
- 动词偏好（own / drive / 搭建 / 落地 / 协同…）
- 隐性信号（"快节奏"=抗压与优先级；"从0到1"=主导性证据）

### 2. 判断叙事主轴
问自己：这个岗位付钱买的核心能力是什么？用一句话写出来。
常见主轴示例：
- 产品岗 → "完整闭环"：需求 → 方案 → 上线 → 验收
- 运营/增长岗 → "数字增长"：覆盖、转化、留存、效率
- 分析/风控岗 → "严谨与口径"：数据定义、验证、异常发现
- 客户向岗 → "翻译能力"：把复杂系统讲给业务方 + 落地采纳

### 3. 重选、重排、重讲
- **重选**：从素材表挑出最贴主轴的 4-6 条，弱相关的删掉（母版是超集，敢删）
- **重排**：最贴 JD 的经历放最前，每段经历内最贴的 bullet 放第一条
- **重讲**：每条 bullet 换成该岗位的语言——事实不变，视角变。关键词自然注入三个位置：summary、每段首条 bullet、技能区。**禁止关键词堆砌**——注入后读起来必须像人话

### 4. 一页检查
定制后必须收进一页 A4。删的顺序：弱相关经历 → 重复证明同一能力的 bullet → 形容词。**数字永远最后删。**

## 红线（重申）
只换表述，永不虚构。JD 要求用户没有的经历/技能：不编，可在 summary 用真实的邻近能力回应，或提示用户这是面试要正面回答的 gap。
