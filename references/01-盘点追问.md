# 第一步：经历输入 + 量化盘点 → 经历素材表

目标：把用户的"流水账"挖成"带数字的价值"。这一步不写简历，只建资产。

## 1. 经历输入（先做这个，不要跳过）

开场主动引导用户把材料全部丢进来，话术示例：

> 在动笔之前，先把你手头一切能反映经历的东西发给我——**旧简历、周报/日报、项目记录、OKR 或绩效自评、老板同事夸你的聊天记录截图、作品或数据看板的链接**……有什么给什么，乱一点没关系，我来整理。什么材料都没有也行，直接跟我说"你做过什么"，我一条条问你。

收材料的原则：
- **宁多勿少**：周报日报里藏着用户自己都忘了的数字，是最好的量化原料
- 材料到手先通读，把候选经历列成清单给用户看（"我从材料里挖到这 N 段经历，还有漏的吗？"），确认覆盖完整再进入追问
- 材料里已有的数字，追问时直接引用求证，不要让用户重复劳动

## 2. 量化追问

### 追问协议（对每一段经历，问到答不出为止）

按顺序追问，每次只问 1-2 个，像朋友聊天不像审讯：

1. **规模**：这个项目/活动多大？（预算、人数、周期、覆盖范围）
2. **角色**：你负责哪一块？是主导还是参与？向谁汇报、带几个人？
3. **动作**：你具体做了什么别人没做的事？（找差异化动作，不要"负责推进"）
4. **结果**：上线/结束后什么数字变了？（增长、转化、效率、成本、错误率）
5. **对比**：这个数字相对之前/同行/目标是什么水平？（没有绝对值就要相对值）
6. **技能**：这件事用到了什么工具、方法、软技能？
7. **证据**：如果面试官追问，你能讲出的细节/故事是什么？

追问技巧：
- 用户说"没有数字"→ 降级量化：频次（每周 N 次）、范围（覆盖 N 个部门）、时长（把 N 天的事压到 N 小时）
- 用户想不起来 → 换角度问："老板/同事因为什么夸过你？""哪件事你交付后别人开始依赖你？"
- 挖到的数字要跟用户确认真实性——**宁可保守，不可放大**

## 输出一：经历素材表

| # | 经历（一句话） | 做了什么（动作） | 结果与数字 | 技能/工具 | 面试可讲的细节 |
|---|---|---|---|---|---|

规则：只加不改；按时间倒序；每条独立可用（不依赖上下文）。存为用户的 `简历素材表.md`。

## 输出二：通用简历母版

从素材表长出一份不针对任何岗位的均衡版：
- 结构：联系方式 → 一句话 summary → 工作经历（每段 3-5 条 bullet）→ 教育 → 技能/加分项
- bullet 公式：**动词开头 + 具体动作 + 量化结果**（"搭建 X，覆盖 N 场景，效率提升 M%"）
- 母版可以略超一页；到第二步按 JD 裁剪时再收进一页
