# 第三步：生成 PDF

目标：内容定稿 → 成品 PDF，排版零手工。

## 模板选择
- 默认：`templates/简历模板-通用.html`（占位符 + 每板块写作公式提示）
- 金融/分析向：参考 `简历示例-金融人-虚构.html` 的成品密度和写法
- 运营/市场向：参考 `简历示例-运营人-虚构.html`
- 版式规格：A4 单页、深青 `#0e7a6e` 强调色、加分项做成徽章样式

## 填充规则
1. 复制模板为用户文件（如 `简历-张三-产品岗.html`），**不改模板原件**
2. 只替换文本节点，不动任何 CSS/结构——版式是写死的，内容是活的。唯一允许调的 CSS 是下面的密度旋钮
3. 删掉示例的"仅供教学演示"页脚声明
4. 每行长度对齐示例的密度：bullet 一行放满但不折两行以上
5. 后续修改：用户说"把某段实习加上/换个说法"→ 只改对应文本节点

## 密度旋钮：让内容正好铺满一页

模板 `:root` 里有且只有一个可调参数，字号与所有间距都按它等比缩放：

```css
--fs: 15.5px;   /* 调这一个数就够，其余样式不要动 */
```

**半页空白比排版丑更伤**——招聘方看到撑不满的一页，第一反应是"这人没什么可写的"。所以填完内容后必须校准：

| 现象 | 怎么调 |
|---|---|
| 底部大片留白（内容撑不满） | 往上调：16px → 16.5px → 17px |
| 溢出到第二页 | 往下调：15px → 14.5px → 14px |
| 调到 14px 还是两页 | 别再调了，回第二步删内容（删的顺序见 02-JD定制：弱相关经历 → 重复的 bullet → 形容词，**数字永远最后删**） |

参考值：内容较多的 4-5 年经验用 15–15.5px；3 年左右、经历较少的用 16.5–17px。

**校准必须看打印结果，不能只看屏幕。** 屏幕上看着刚好铺满的，打印时常常正好挤出第二页。每次调完都要重新导一次 PDF 确认页数（导出方式见下），目标是**填充到页面 88–93% 且仍为 1 页**。

## 导出 PDF
两种方式（都验证过）：
- **手动**：浏览器打开 HTML → 打印（Cmd+P）→ 存为 PDF。设置：A4、边距默认、**不勾页眉页脚**、缩放 100%
- **命令行**（macOS Chrome）：
  ```bash
  "/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" \
    --headless --print-to-pdf="输出.pdf" --no-pdf-header-footer "file://简历.html"
  ```

## 出品检查
- [ ] 一页 A4，无第二页
- [ ] 内容铺满 88–93%，底部没有大片空白（不满就调 `--fs`）
- [ ] 每段经历首条 bullet 有数字
- [ ] 教学声明已移除
- [ ] 联系方式真实无误
- [ ] PDF 文件名规范：`姓名-岗位方向-简历.pdf`
